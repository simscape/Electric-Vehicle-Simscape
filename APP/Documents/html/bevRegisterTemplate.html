bevRegisterTemplate

Register a vehicle template by scanning its model

Syntax

configEntry = bevRegisterTemplate(templatePath)
configEntry = bevRegisterTemplate(templatePath, Name=Value)
bevRegisterTemplate(___)

Description

configEntry = bevRegisterTemplate(templatePath) scans the vehicle template .slx file for subsystem reference (SSR) blocks, maps each referenced model to a component folder under Components/, groups instances by component type, and returns a config struct compatible with VehicleTemplateConfig.json. By default, this is a dry run — no file is written.

configEntry = bevRegisterTemplate(templatePath, Name=Value) specifies additional options using one or more name-value arguments. Use ConfigFile and DryRun=false to write the entry to a JSON config file.

The config key is always the .slx model basename. For example, VehicleThreeWheel.slx registers as "VehicleThreeWheel". Different fidelity presets for the same template go in separate config JSON files.

Examples

expand all collapse all
Dry Run — Preview Config Entry

Scan a template and display the config summary without writing any file.

entry = bevRegisterTemplate("Model/VehicleTemplate/VehicleThreeWheel.slx")

The function prints preset status (which config files already contain this template), the SSR block scan results, and a component/instance/model summary table.

Register to a New Config File

Scan the template and write the entry to a new JSON file in APP/Config/Preset/.

bevRegisterTemplate("Model/VehicleTemplate/VehicleThreeWheel.slx", ...
    Description     = "Three-wheel electric vehicle", ...
    ConfigFile      = "ThreeWheelConfig.json", ...
    SystemParameter = "NA", ...
    DryRun          = false)

The config key "VehicleThreeWheel" is derived from the .slx filename automatically.

Register to an Existing Config File

Add the template entry alongside existing templates. If the template already exists in the file, you are prompted to confirm overwrite.

bevRegisterTemplate("Model/VehicleTemplate/VehicleThreeWheel.slx", ...
    ConfigFile = "VehicleTemplateConfig.json", ...
    DryRun     = false)
Create Multiple Fidelity Presets

Register the same template into separate config files to create different fidelity presets. The app's Config dropdown lets users switch between them.

% Default fidelity preset
bevRegisterTemplate("Model/VehicleTemplate/VehicleElectric.slx", ...
    Description = "Standard fidelity", ...
    ConfigFile  = "VehicleTemplateConfig.json", ...
    DryRun      = false)

% High-fidelity thermal preset
bevRegisterTemplate("Model/VehicleTemplate/VehicleElectric.slx", ...
    Description     = "Thermal fidelity preset", ...
    ConfigFile      = "VehicleThermalConfig.json", ...
    SystemParameter = "BEVThermalParams", ...
    DryRun          = false)

Then use bevAddFidelity to customize the models in each preset.

Input Arguments

templatePath — Path to vehicle template

Path to the vehicle template .slx file, specified as a string scalar. Can be an absolute path or a path relative to the project root.

The model basename becomes the config key (e.g., "VehicleThreeWheel.slx" → config key "VehicleThreeWheel").

Data Types: string

Name-Value Arguments

Specify optional pairs of arguments as Name=Value.

Description — Short description string

One-line description stored in the config entry. Appears in the app UI and generated README files.

Default: "" (no description)

Data Types: string

ConfigFile — JSON config filename

Name of the JSON file to write to. A bare filename (no path separator) resolves to APP/Config/Preset/. A path is used as-is or resolved relative to the project root.

Default: "" (no file — dry run only)

Data Types: string

DryRun — Preview mode

If true, prints a summary but does not write any file. If false and ConfigFile is specified, writes the config. If false and ConfigFile is empty, prints a hint showing the required syntax.

Default: true

Data Types: logical

SystemParameter — System-level parameter files

Names of system-level parameter files required by this template (e.g., "BEVThermalParams"). Use "NA" if none.

Default: "NA"

Data Types: string array

ProjectRoot — Project root folder

Override the project root instead of auto-detecting via matlab.project.rootProject().

Default: "" (auto-detect)

Data Types: string

Output Arguments

configEntry — Config struct for the template

Struct matching the VehicleTemplateConfig.json schema with fields:

FieldTypeDescription
DescriptioncharShort description (if provided)
ComponentsstructComponent types, each with Instances and Models
ControlsstructController instances and models (if detected)
SystemParametercellSystem parameter file names

Tips

  • The function checks all *.json files in APP/Config/Preset/ and reports which ones already contain the template before scanning.
  • SSR blocks that reference models outside Components/ are reported as unmapped but do not cause an error.
  • Controller components (under Components/Controller/) are automatically placed in the Controls section, not Components.
  • The config is validated with validateVehicleConfig before writing. If validation fails, no file is written.
  • To create multiple fidelity presets for the same template, register into separate config files. The app's Config dropdown lets users switch between them.
