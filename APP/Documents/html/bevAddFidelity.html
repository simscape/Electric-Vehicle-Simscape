bevAddFidelity

Add a model fidelity to an existing template config

Syntax

bevAddFidelity(configFile, templateName)
bevAddFidelity(configFile, templateName, componentName)
bevAddFidelity(configFile, templateName, componentName, modelName)
bevAddFidelity(___, Name=Value)

Description

bevAddFidelity(configFile, templateName) lists all components in the specified template, showing instances and current models for each. No changes are made.

bevAddFidelity(configFile, templateName, componentName) lists all models available on disk for the specified component, marking which are already in the config and which can be added.

bevAddFidelity(configFile, templateName, componentName, modelName) adds the model to the component's Models list. By default this is a dry run. Use DryRun=false to write.

bevAddFidelity(___, Name=Value) specifies additional options. Use ParamFile to link a non-convention parameter file by creating a Selections section (same structure the app uses in saved user setups). Use MakeDefault=true to place the model first in the list (the app default).

Examples

expand all collapse all
List Components in a Template

See what components a template contains before deciding which to modify.

bevAddFidelity("VehicleTemplateConfig.json", "VehicleElectric")
Components in VehicleElectric: Component Instances Models -------------------- ----------------------------------- ----------------------------- BatteryHV Battery [BatteryLumped] MotorDrive Rear Motor (EM2), Front Motor (EM1) [MotorDriveGear] Charger Charger [Charger, ChargerDummy] Driveline Driveline [Driveline, DrivelineWithBraking] Tip: bevAddFidelity(configFile, "VehicleElectric", "<ComponentName>") to see available models for a component.
List Available Models for a Component

See which fidelities exist on disk for a component and which are already registered.

bevAddFidelity("VehicleTemplateConfig.json", "VehicleElectric", "MotorDrive")
MotorDrive in VehicleElectric: Instances: Rear Motor (EM2), Front Motor (EM1) Current Models: [MotorDriveGear] Available on disk (Components/MotorDrive/Model/): MotorDriveGear in config MotorDriveGearTh can add MotorDriveLube can add Tip: bevAddFidelity(configFile, "VehicleElectric", "MotorDrive", "<ModelName>") to add a model fidelity.
Add a Fidelity (Dry Run)

Preview what the addition looks like before committing to disk. By default DryRun is true.

bevAddFidelity("VehicleTemplateConfig.json", ...
    "VehicleElectric", "MotorDrive", "MotorDriveGearTh")
Add with Custom Param File

When the default <ModelName>Params.m convention does not apply (e.g., reusing a 4-wheel model in a 3-wheel template with different parameters), specify ParamFile to create a Selections section linking the param file for all instances.

bevAddFidelity("VehicleTemplateConfig.json", ...
    "VehicleElectric", "MotorDrive", "MotorDriveGearTh", ...
    ParamFile   = "ThreeWheelMotorParams.m", ...
    MakeDefault = true, ...
    DryRun      = false)

This produces a component entry with Selections:

"MotorDrive": {
    "Instances": ["Rear Motor (EM2)", "Front Motor (EM1)"],
    "Models": ["MotorDriveGearTh", "MotorDriveGear"],
    "Selections": {
        "RearMotor_EM2_": {"Label": "Rear Motor (EM2)", "Model": "MotorDriveGearTh", "ParamFile": "ThreeWheelMotorParams.m"},
        "FrontMotor_EM1_": {"Label": "Front Motor (EM1)", "Model": "MotorDriveGearTh", "ParamFile": "ThreeWheelMotorParams.m"}
    }
}
Make a Fidelity the Default

Use MakeDefault=true to place the model first in the Models array. The app selects the first valid model as the dropdown default.

bevAddFidelity("VehicleTemplateConfig.json", ...
    "VehicleElectric", "Charger", "ChargerThermal", ...
    MakeDefault = true, ...
    DryRun      = false)
Modify an Existing Fidelity

If a model is already in the config, you can still modify it. Specify ParamFile to link a different parameter file, or MakeDefault=true to move it to the default position. The model is not duplicated.

% Link a new param file to an existing fidelity
bevAddFidelity("VehicleTemplateConfig.json", ...
    "VehicleElectric", "MotorDrive", "MotorDriveGear", ...
    ParamFile = "ThreeWheelMotorParams.m", ...
    DryRun    = false)

% Move an existing fidelity to the default position
bevAddFidelity("VehicleTemplateConfig.json", ...
    "VehicleElectric", "Charger", "ChargerDummy", ...
    MakeDefault = true, ...
    DryRun      = false)

Input Arguments

configFile — JSON config filename

Name of the JSON config file, specified as a string scalar. A bare filename (no path separator) resolves to APP/Config/Preset/.

Data Types: string

templateName — Template key in config

Top-level key in the JSON config that identifies the template (e.g., "VehicleElectric"). Must match the .slx model basename.

Data Types: string

componentName — Component type

Name of the component type within the template (e.g., "MotorDrive", "BatteryHV"). Omit to list all components (Level 1 discovery).

Default: ""

Data Types: string

modelName — Model basename to add

Basename of the .slx model file to add to the component's Models list. Omit to list available models on disk (Level 2 discovery).

Default: ""

Data Types: string

Name-Value Arguments

ParamFile — Parameter file to link

Filename of the parameter script to link to this fidelity. When specified, a Selections section is created for all instances of the component, using the same structure as app-saved user setups. The app's applySelections function handles this directly on preset load.

If omitted, the app uses convention-based resolution (<ModelName>Params.m).

Default: ""

Data Types: string

MakeDefault — Set as default fidelity

If true, the model is placed first in the Models array. The app selects the first valid model as the default in the dropdown.

Default: false

Data Types: logical

DryRun — Preview mode

If true, prints a summary without writing. If false, writes the modified config to disk after validation.

Default: true

Data Types: logical

ProjectRoot — Project root folder

Override the project root instead of auto-detecting.

Default: "" (auto-detect)

Data Types: string

Tips

  • Use progressive discovery: start with just configFile and templateName, then narrow down by adding componentName, then modelName. Each level shows a hint for the next call.
  • The Selections section uses matlab.lang.makeValidName for instance keys, matching the app's convention in buildSetupState.
  • A duplicate model (already in the Models list) is detected and skipped with a message. No error is thrown.
  • The config is validated with validateVehicleConfig before writing. If the modified config is structurally invalid, no file is written.
  • To create a different fidelity preset for the same template, register the template into a separate config file with bevRegisterTemplate, then use this function to customize it.
  • The model .slx is checked against Components/ folders on disk. A warning is shown if the model is not found, but the addition proceeds.
  • When ParamFile is specified, the file is checked in the component's Model/ folder. A note is shown if it does not exist yet.
