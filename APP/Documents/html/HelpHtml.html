The BEV App lets you interactively create and configure Battery Electric Vehicle (BEV) models using Simscape. The app streamlines the process of assembling a complete electric vehicle architecture, so you can focus on design and analysis instead of manual model setup.

Vehicle Platform (top-left)

BEV Model
Select the Simulink model (.slx) that already contains the BEV architecture canvas. The app will apply your selections to this model.
Veh Platform
Choose the vehicle type and fidelity:
  • Vehicle type: Car, SUV, etc.
  • Fidelity: with/without Thermal/HVAC, and level of detail (number/kind of components used).
Config
Pick a JSON configuration that defines which components and fidelity options are allowed for the selected platform.
The app uses this JSON to control what shows up later in the Components dropdowns.

Components (left column)

For each subsystem (e.g., BatteryHV, Front Motor, Rear Motor, etc.), the app builds the dropdown on the fly by reading:
  • Your JSON Config (allowed items), and
  • The folder names in your repo (available implementations).
Actions per component
  • Open: Opens the selected component’s model.
  • Param: Opens the associated parameter file for that component.
Tip: If you don’t see a component option you expect, confirm it’s (a) listed in the JSON and (b) present in the expected repo folder. Selection Preview (center-top)
A dynamic canvas screenshot of the BEV model that updates whenever you change selections.
  • Use the “?” button to manually refresh the preview on demand.

Description (center-bottom)

Shows contextual details for the currently selected platform/component to guide your choices.

Env. Condition (top-right)

Set global ambient conditions (temperature, pressure, humidity, CO₂ fraction).
These values are applied across the entire model via Simscape Foundation environment settings, so every thermal/airflow calculation sees the same environment.

HVAC (middle-right)

Available when you chose a platform with Thermal/HVAC.
  • Add an HVAC load to the model.
  • Toggle A/C ON/OFF and set Cabin Temperature—just like a real vehicle’s climate control.

Driver Setup (bottom-right)

Provide control parameters used inside the model to activate features:
  • AWD (all-wheel drive)
  • Regen (regenerative braking)
  • Charging (include charging behavior)
  • Drive Cycle (select the operating profile)
These switches set control parameters that the model consumes to enable the corresponding behaviors.

Model Actions (bottom-right corner)

  • Model Tree: Inspect the structure/hierarchy of the configured model.
  • Config Export: Save your current selections to a JSON for repeatable runs and sharing.
  • Create Model: Apply your selections to the chosen .slx (and generate/update the configured model so it’s ready to simulate).

Quick Start (3 steps)

  1. Pick your canvas: Choose the .slx (BEV Model), then select Veh Platform and the Config JSON.
  2. Assemble: For each Component, choose an implementation; adjust Env. Condition, HVAC, and Driver Setup.
  3. Build: Check Selection Preview, export Config if needed, then click Create Model to finalize.
