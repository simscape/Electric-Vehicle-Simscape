Configure a BEV Model Using the BEV App

This workflow shows how to configure a battery electric vehicle model using the BEV App. You will select a vehicle platform, choose component models and controls, set environment and HVAC options, configure the driver and drive cycle, save your setup, and create a Simulink model with parameter scripts.

Overview

The BEV App provides a guided interface for assembling a battery electric vehicle model from component subsystems. Instead of manually wiring models and editing scripts, you select a vehicle configuration, pick component fidelities from dropdowns, set environment and HVAC conditions, and let the app generate the configured model and parameter scripts.

Prerequisites

Before starting, open the project by running the project file.

openProject("ElectricVehicleSimscape.prj")

Opening the project initializes the environment and makes all models, component folders, configuration files, and helper scripts available to the app.

Required products

  • MATLAB
  • Simulink
  • Simscape
  • Simscape Electrical
  • Simscape Driveline

Optional

  • Simscape Battery
  • Simscape Fluids
Keep the base SLX model under version control and create a copy for experiments.

Select a Vehicle Platform

Select a base model and a vehicle configuration so the app can determine which component and control options are available.

The following list describes what each dropdown does.

  1. Base Model — select the .slx canvas that contains the BEV layout.
  2. Design Scenario — select the JSON config that defines allowed components and fidelities. Preset configs ship with the project; user-saved setups appear below a separator.
  3. Veh. Config — select the vehicle template that defines the model architecture.

Choose Component Models

Select the component variants used to build the configured model. The app reads the design scenario JSON and matches each instance in the base model to its list of allowed component models.

The label in each row comes from the config Instances list and must match a subsystem name in your base model. The dropdown lists model files defined under Models, filtered to show only models that exist on disk.

How the dropdown gets populated

  • The app scans the component repository for available model files.
  • It filters them against the config Models list for each component.
  • Only matching models appear in the dropdown. Missing models are marked with a red note.

Example config structure:

"Components": {
  "BatteryHV": {
    "Instances": ["Battery"],
    "Models": ["BatteryLumped", "BatteryLumpedThermal", "BatteryTableBased"]
  },
  "MotorDrive": {
    "Instances": ["Rear Motor (EM2)", "Front Motor (EM1)"],
    "Models": ["MotorDriveGear", "MotorDriveGearTh", "MotorDriveLube"]
  }
}

Controls in each component row

ControlDescription
LabelInstance name from config Instances; must match the subsystem name in the base model.
DropdownSelect a component model from the filtered Models list.
OpenOpen the selected component model in Simulink.
ParamOpen the parameter script for the component. Right-click to link or unlink a custom param file.

Link parameter files

Link a parameter file to a component so the Param button opens the correct script during setup and export. The default convention is <ModelName>Params.m in the component's Model folder. Right-click the Param button to link or unlink a custom file.

Start with nonthermal variants to check connectivity, then switch to electro-thermal variants after the setup is stable.

Review the Preview and Description

Confirm that the selected component model matches what you intend to use. The preview and description update automatically when you change a dropdown selection.

Preview panel

  • Shows a snapshot of the selected model canvas.
  • Updates when you change a dropdown selection.
  • Helps catch missing connections or wrong instance selection before creating the model.

Description panel

  • Summarizes the purpose of the selected model.
  • Describes main interfaces and key assumptions.
  • Provides context when multiple fidelity variants exist.

Define Environment and Initial Conditions

Define ambient environment and initial state values used by thermal and fluid components. These values propagate to the environment subsystem in the base model.

ParameterDescriptionDefault
Amb. TempAmbient air temperature around the vehicle25 °C
Amb. Press InitInitial atmospheric pressure1 bar
Rel. Humidity InitInitial relative humidity of ambient air0.3
CO2 Fraction InitCarbon dioxide volume fraction in ambient air0.0004
Define multiple environment presets (Hot City, Cold Highway, Altitude Test), then compare results by switching presets.

Configure HVAC Settings

Control whether cabin air conditioning is active during simulation and set the target cabin temperature. This panel is enabled only when the selected configuration includes an HVAC component.

ControlDescriptionDefault
AC StatusToggle the air conditioning system on or offOff
Cabin Temp SetpointDesired cabin air temperature20 °C

Turning AC on increases total power demand and thermal load, which affects battery drain and range. For baseline studies, run with AC off first, then enable AC to measure the impact on efficiency.

Set Up Driver, Controls, and Drive Cycle

Set drive mode options, select a controller model, and choose a drive cycle. These settings define how the vehicle is driven in simulation and which speed profile the driver follows.

Drive mode buttons

ButtonDescription
AWDEnable or disable all-wheel drive. When enabled, both front and rear motors are active.
RegenEnable or disable regenerative braking. When disabled, the model needs a mechanical brake subsystem.
ChargingSwitch to charging mode. Disables propulsion settings such as AWD and Regen.

Control selection

The controller dropdown is populated from the config Controls section. It lists control models available for the selected configuration. 

"Controls": {
  "Instances": ["Controller"],
  "Models": ["ControllerFRM", "ControllerHVAC"]
}
ElementDescription
DropdownLists control models from config Models.
?Shows the control description and preview of the control block layout.

Drive cycle selection

The drive cycle dropdown defines which speed-versus-time profile the driver follows. The list is populated from options available in the drive cycle source block inside the base model.

  • Common cycles: FTP72, WLTP, NEDC, US06.
  • If no options appear, install the Simulink Support Package for Vehicle Drive Cycles.
  • The ? button opens the description and preview of the selected drive cycle.
Start with a short cycle such as FTP72 to verify setup, then switch to longer cycles such as WLTP for performance studies.

Save Setup, Export Parameters, and Create Model

In this section, you save your configuration, generate parameter scripts, and create the Simulink model.

Save Setup

Save the current session as a JSON config file. The saved setup captures all component selections, environment settings, HVAC conditions, and operating modes.

  • Saves to APP/Config/User/ as a JSON file.
  • The saved setup appears in the Design Scenario dropdown below a separator.
  • Loading a saved setup restores the full session including all selections and settings.
Save Setup creates a reusable JSON config that can be loaded back into the app. Parameter Export generates a MATLAB script for simulation. They serve different purposes.

Parameter Export

Generate MATLAB parameter scripts that correspond to the selected components and controls.

  • Creates or updates a parameter script for the current configuration.
  • Collects linked component parameter files from the Components panel.
  • Writes HVAC, environment, and driver variables when exposed by the configuration.

Missing parameter link panel

If components do not have a parameter file linked, the app opens a link panel when you press Parameter Export.

  • Browse to an existing parameter file and link it.
  • Continue with defaults for components that remain unlinked.

This uses the same linking mechanism as the right-click option in the Components panel.

Create Model

Generate or update the configured Simulink model so it is ready to simulate.

  • Opens the selected base model.
  • Locates subsystem instances defined by config Instances.
  • Replaces or configures those subsystems with the selected component models.
  • Applies environment, HVAC, driver, controls, and drive cycle settings.
