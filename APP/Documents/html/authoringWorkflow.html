Add Templates and Fidelities to the BEV App

Register your own vehicle templates and model fidelities so the app can use them

Overview

The BEV App loads vehicle templates and their available component fidelities from config files in APP/Config/Preset/. Two command-line functions let you register new content without editing those files by hand.

TermMeaning
Template A top-level vehicle .slx model containing subsystem references to components.
Component A reusable subsystem type (e.g., MotorDrive, BatteryHV) under Components/.
Fidelity A specific model variant for a component (e.g., MotorDriveGear vs MotorDriveGearTh).
Preset A config file defining which fidelities are available. Different presets can offer different choices for the same template.

Both functions default to dry run mode — preview first, then set DryRun=false to write.

Register a Template

Use bevRegisterTemplate to scan a vehicle template .slx and write its component structure to a config file.

1. Scan the Template

Run a dry run to see what the CLI detects. The function loads the model, finds all subsystem reference blocks, maps each to a Components/ folder, and prints a summary.

bevRegisterTemplate("Model/VehicleTemplate/VehicleThreeWheel.slx")

Review the output to confirm all components and instances were detected correctly.

2. Write to a Config File

When the scan looks correct, specify ConfigFile and set DryRun=false.

bevRegisterTemplate("Model/VehicleTemplate/VehicleThreeWheel.slx", ...
    Description = "Three-wheel electric vehicle", ...
    ConfigFile  = "ThreeWheelConfig.json", ...
    DryRun      = false)

The config key is derived from the .slx filename automatically. If the template already exists in the target file, an error is thrown. Use Overwrite=true to replace the existing entry.

3. Verify in the App

Open the BEV App, select your config from the Config dropdown, and confirm the template appears with the correct component dropdowns.

Add Fidelities

After registration, the config lists only the fidelities found at scan time. Use bevAddFidelity to add more.

1. Explore Components and Models

Call the function with fewer arguments to discover what is available.

% List all components in the template
bevAddFidelity("ThreeWheelConfig.json", "VehicleThreeWheel")

% List models on disk for a specific component
bevAddFidelity("ThreeWheelConfig.json", "VehicleThreeWheel", "MotorDrive")

The output marks which models are already in the config and which can be added.

2. Add a Model

Specify all four positional arguments to add a fidelity.

bevAddFidelity("ThreeWheelConfig.json", "VehicleThreeWheel", ...
    "MotorDrive", "MotorDriveGearTh", DryRun = false)

To make the new fidelity the default selection in the app dropdown, use MakeDefault=true — this places the model first in the list.

3. Link a Parameter File (Optional)

By default, the app resolves parameter files by convention (<ModelName>Params.m). When this does not apply — for example, when reusing a component model with different parameters — specify ParamFile to link a file explicitly.

bevAddFidelity("ThreeWheelConfig.json", "VehicleThreeWheel", ...
    "MotorDrive", "MotorDriveGearTh", ...
    ParamFile = "ThreeWheelMotorParams.m", ...
    DryRun    = false)

4. Verify in the App

Open the BEV App, select the template, and confirm the new fidelity appears in the component dropdown. Select it and verify the model loads without errors.

Add a Component

When you add a new component to a template model (for example, adding a HeatPump subsystem reference to VehicleElectric.slx), use bevUpdateTemplate to sync all configs that reference that template.

1. Modify the Template Model

Add the new subsystem reference block to your template .slx in Simulink. Make sure it points to a model under Components/.

2. Preview the Sync

Run a dry run to see what changed. The function finds every config file containing this template and prints a diff.

bevUpdateTemplate("Model/VehicleTemplate/VehicleElectric.slx")

The output shows new components as +, removed as -, and modified instances as ~.

3. Apply Changes

bevUpdateTemplate("Model/VehicleTemplate/VehicleElectric.slx", ...
    DryRun = false)

New components are added with the referenced model only. Existing fidelities and Selections in other components are preserved. Use bevAddFidelity afterwards to add extra fidelities for the new component.

4. Verify in the App

Open the BEV App, select the template, and confirm the new component dropdown appears.

Config Cleanup

Over time, model .slx files may be renamed or deleted from Components/*/Model/. Use bevCleanConfig to remove stale entries from a config file.

1. Preview Stale Entries

bevCleanConfig("VehicleTemplateConfig.json")

The function checks every model in the config against what exists on disk and prints a per-template report.

2. Remove Stale Entries

bevCleanConfig("VehicleTemplateConfig.json", DryRun = false)

Models whose .slx files are missing are removed. If all models for a component are stale, the entire component is removed. Selections referencing removed models are cleaned up automatically.

Multiple Presets

To offer different fidelity choices for the same template, register it into separate config files. Each file acts as a preset that appears in the app's Config dropdown.

% Standard preset
bevRegisterTemplate("Model/VehicleTemplate/VehicleElectric.slx", ...
    ConfigFile = "StandardConfig.json", DryRun = false)

% Thermal preset — register, then swap in thermal fidelities
bevRegisterTemplate("Model/VehicleTemplate/VehicleElectric.slx", ...
    ConfigFile = "ThermalConfig.json", DryRun = false)

bevAddFidelity("ThermalConfig.json", "VehicleElectric", ...
    "MotorDrive", "MotorDriveGearTh", MakeDefault = true, DryRun = false)

bevAddFidelity("ThermalConfig.json", "VehicleElectric", ...
    "BatteryHV", "BatteryLumpedTh", MakeDefault = true, DryRun = false)
