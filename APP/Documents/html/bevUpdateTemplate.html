bevUpdateTemplate

Rescan a template and sync all configs that reference it

Syntax

bevUpdateTemplate(templatePath)
bevUpdateTemplate(templatePath, Name=Value)

Description

bevUpdateTemplate(templatePath) rescans the template .slx for subsystem reference blocks, finds every config file in APP/Config/Preset/ that contains this template, and prints a diff showing what changed. By default this is a dry run.

bevUpdateTemplate(templatePath, DryRun=false) applies the changes to each config file after validation.

Use this after modifying a template model — for example, after adding a new component like HeatPump or renaming an instance block. The function detects what was added, removed, or changed and updates every config that references the template.

Examples

expand all collapse all
Preview After Adding a Component

You added a HeatPump subsystem reference to VehicleElectric.slx. Preview the diff before writing.

bevUpdateTemplate("Model/VehicleTemplate/VehicleElectric.slx")

The output lists each config file that contains VehicleElectric and shows what would change: + HeatPump for the new component.

Apply Changes to All Configs

After reviewing the diff, apply the changes.

bevUpdateTemplate("Model/VehicleTemplate/VehicleElectric.slx", ...
    DryRun = false)

Each config file is validated before writing. Existing fidelities and Selections added via bevAddFidelity are preserved.

Sync After Removing a Component

If you removed the Charger subsystem reference from the template, the function detects the removal and deletes it from each config.

bevUpdateTemplate("Model/VehicleTemplate/VehicleElectric.slx", ...
    DryRun = false)

The output shows - Charger (removed from template) for each affected config.

Sync After Renaming an Instance

If you renamed a block from Rear Motor to Rear Drive Unit, the function updates instances in all configs. Selections for the old instance name are cleaned up automatically.

bevUpdateTemplate("Model/VehicleTemplate/VehicleElectric.slx", ...
    DryRun = false)

Input Arguments

templatePath — Path to vehicle template

Path to the vehicle template .slx file, specified as a string scalar. Can be an absolute path or a path relative to the project root.

Data Types: string

Name-Value Arguments

DryRun — Preview mode

If true, prints a diff for each config but does not write. If false, applies changes after validation.

Default: true

Data Types: logical

ProjectRoot — Project root folder

Override the project root instead of auto-detecting.

Default: "" (auto-detect)

Data Types: string

Sync Rules

The template scan is the source of truth for structure. User-added content (extra fidelities, Selections, parameter links) is preserved.

SituationAction
Component in config and template, unchanged Kept as-is (all fidelities and Selections preserved)
Component in config and template, instances changed Instances updated from template. Selections for removed instances cleaned up.
Component in config and template, new SSR model SSR model added to the models list. Existing fidelities kept.
Component in config but not in template Removed from config
Component in template but not in config Added with instances and SSR model only

Tips

  • Always preview with a dry run before writing. The diff output shows exactly what will change in each config file.
  • The function searches all *.json files in APP/Config/Preset/. If the template is not found in any config, use bevRegisterTemplate first.
  • New components are added with only the SSR-linked model. Use bevAddFidelity to add extra fidelities and parameter links afterwards.
  • Description, SystemParameter, and other metadata fields are not touched by the sync — only the structural component/instance/model data.
  • The Controls section follows the same sync rules as Components.
