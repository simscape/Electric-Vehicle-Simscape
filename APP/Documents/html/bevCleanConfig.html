bevCleanConfig

Remove stale fidelities from a config file

Syntax

bevCleanConfig(configFile)
bevCleanConfig(configFile, Name=Value)

Description

bevCleanConfig(configFile) scans a config JSON and checks each model in every Models list against the Components/*/Model/ folders on disk. Models whose .slx files no longer exist are flagged as stale. By default this is a dry run — stale entries are listed but the file is not modified.

bevCleanConfig(configFile, DryRun=false) removes stale models from the config and writes the cleaned file. If all models for a component are stale, the entire component entry is removed. Selections referencing removed models are cleaned up automatically.

Use this after deleting or renaming model .slx files to keep configs in sync with what actually exists on disk.

Examples

expand all collapse all
Preview Stale Entries

Scan a config and see which models are stale without writing anything.

bevCleanConfig("VehicleTemplateConfig.json")
Template: VehicleElectric - Charger.ChargerOld (not found on disk) Template: VehicleElecAux (no stale entries) ============================================ Cleanup Summary ============================================ Config: APP/Config/Preset/VehicleTemplateConfig.json Removed: 1 stale model(s) VehicleElectric: BatteryHV [BatteryLumped] Charger [Charger, ChargerDummy] MotorDrive [MotorDriveGear] DryRun = true. No file written.
Remove Stale Entries

Apply the cleanup after reviewing the preview.

bevCleanConfig("VehicleTemplateConfig.json", DryRun = false)

The config is validated before writing. The original file is preserved if validation fails.

Component Fully Removed

When all models for a component are stale, the entire component is removed.

Template: VehicleElectric - HeatPump.HeatPumpOld (not found on disk) * HeatPump removed entirely (no models left)
Selections Cleaned Up

Selections referencing a removed model are cleaned automatically. Selections for surviving models are kept intact.

Template: VehicleElectric - MotorDrive.MotorDriveOld (not found on disk) Selections.RearMotor cleaned (model MotorDriveOld removed) ============================================ Cleanup Summary ============================================ Config: APP/Config/Preset/VehicleTemplateConfig.json Removed: 1 stale model(s) VehicleElectric: MotorDrive [MotorDriveGear, MotorDriveGearTh]
Clean Config — No Changes

If all models exist on disk, the function reports a clean config.

bevCleanConfig("VehicleTemplateConfig.json")
Template: VehicleElectric (no stale entries) Template: VehicleElecAux (no stale entries) Config is clean. No stale models found.

Input Arguments

configFile — JSON config filename

Name of the JSON config file, specified as a string scalar. A bare filename (no path separator) resolves to APP/Config/Preset/. A full or relative path is used as-is.

Data Types: string

Name-Value Arguments

DryRun — Preview mode

If true, prints a report of stale entries without modifying the file. If false, removes stale entries and writes the cleaned config.

Default: true

Data Types: logical

ProjectRoot — Project root folder

Override the project root instead of auto-detecting.

Default: "" (auto-detect)

Data Types: string

Cleanup Rules

Each model in Models is checked against the Components/*/Model/ folders on disk using the same lookup as bevAddFidelity.

Cleanup behavior

SituationAction
Model .slx exists on disk Kept in config
Model .slx not found on disk Removed from Models list
All models for a component are stale Entire component entry removed
Selection references a removed model Selection entry removed
Controls models stale Stale models removed; Controls section removed if empty

Tips

  • Always preview with a dry run first. The per-template report shows exactly which models and Selections would be removed.
  • The cleanup summary includes the cleaned state after removal, so you can verify the remaining config is correct before writing.
  • If the config file is not found or empty, the function returns gracefully with a message.
  • The cleaned config is validated with validateVehicleConfig before writing. If validation fails, no file is written.
  • Use this after removing .slx files from Components/*/Model/. The function is the inverse of bevAddFidelity — it removes what no longer exists instead of adding what is new.
